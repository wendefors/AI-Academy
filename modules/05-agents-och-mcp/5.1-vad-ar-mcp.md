# Vad är MCP?

MCP (Model Context Protocol) är ett protokoll som gör det möjligt för AI-modeller att kommunicera med externa verktyg och system på ett standardiserat sätt.

Det låter tekniskt, och det är det, men konsekvenserna av det är enkla att förstå: MCP är det som gör att AI kan göra saker i världen, inte bara svara i ett chatfönster.

---

## Problemet som MCP löser

En AI-modell utan verktyg kan bara läsa text du ger den och producera text tillbaka. Det är kraftfullt för textarbete. Men det är isolerat.

Tänk på skillnaden:
- **AI i en chat:** Du klistrar in innehållet manuellt. AI svarar. Du kopierar resultatet manuellt.
- **AI med verktyg:** AI kan läsa en fil direkt. Köra en sökning. Skicka en förfrågan till en tjänst. Skriva tillbaka till ett dokument.

Varje steg som tidigare krävde att du manuellt hämtade och levererade information kan ersättas med ett verktygsanrop.

MCP är standarden som beskriver hur det samtalet mellan AI och verktyg ska gå till, på ett sätt som fungerar oavsett vilket verktyg eller vilken AI-modell.

---

## Tool use: det grundläggande konceptet

Innan MCP existerade verktygsintegration, men utan standard. Varje system implementerade det på sitt eget sätt.

MCP standardiserar det. En MCP-server exponerar verktyg som AI kan anropa. AI-modellen vet vilka verktyg som finns tillgängliga, vad de tar som input, och vad de returnerar.

Exempel på verktyg som kan exponeras via MCP:
- Läsa filer från en mapp
- Söka i en databas
- Göra en webbsökning
- Skicka ett mejl
- Skapa ett kalenderinlägg
- Hämta data från ett API

Var och en av dessa är ett anrop som AI kan göra när det behövs för att lösa uppgiften.

---

## Integrationslager

MCP kan ses som ett integrationslager: ett standardiserat sätt att koppla ihop AI med de system och datakällor som redan finns i din organisation.

Det innebär att AI inte längre behöver vara ett isolerat verktyg. Det kan vara en del av dina befintliga flöden.

Exempel: Du har ett system för kundärenden. Med MCP-integration kan AI läsa inkommande ärenden, koppla dem till relevant historik, förbereda ett svarsutkast, och flagga dem till en handläggare, utan att du manuellt kopierar information fram och tillbaka.

---

## Var det befinner sig idag

MCP är relativt nytt (standardiserat av Anthropic 2024) men har fått snabb adoption. Allt fler AI-verktyg stödjer det, och allt fler tjänster bygger MCP-servrar som exponerar deras funktioner.

Det är inte ett moget, stabilt ekosystem ännu. Det förändras snabbt. Men riktningen är tydlig: det pågår ett skifte från AI som isolerat textverktyg till AI som en aktiv komponent i arbetsflöden.

Det är värt att förstå det konceptuellt nu, även om de tekniska detaljerna fortfarande förändras.

## Labb

**Mål:** Förstå och testa tool use i praktiken.

Det enklaste sättet att uppleva MCP-liknande funktionalitet utan att sätta upp en server är att använda ett AI-verktyg som redan har verktygsintegration inbyggd: till exempel Claude Code, ChatGPT med verktyg, eller liknande.

1. Välj ett AI-verktyg som kan läsa filer eller göra webbsökningar
2. Ge det en uppgift som kräver access till externa resurser: till exempel att läsa en specifik fil och sammanfatta den, eller söka efter aktuell information om ett ämne
3. Observera hur AI använder verktyget: vad anropade det, vad fick det tillbaka, hur påverkade det svaret?
4. Jämför: Hur skiljer det sig från att du manuellt hade hämtat informationen och klistrat in den?

---

## Sammanfattning

- MCP är ett protokoll som gör att AI kan kommunicera med externa verktyg och system
- Tool use gör det möjligt för AI att göra saker i världen, inte bara svara på text
- MCP standardiserar det här på ett sätt som fungerar över modeller och verktyg
- Det är i snabb utveckling: det konceptuella är stabilt, de tekniska detaljerna förändras

---

## Kom igång

- Prova ett AI-verktyg med inbyggda verktyg (fil-access, webbsökning)
- Observera hur verktygsanrop påverkar konversationens kapabilitet
- Identifiera ett arbetsflöde där en enkel verktygskoppling hade sparat tid
